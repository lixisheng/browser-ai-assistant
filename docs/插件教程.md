# Browser AI Assistant 插件教程

本文档面向插件使用者和项目维护者，说明 Browser AI Assistant 的安装、配置、核心功能、常见使用流程和维护注意事项。

## 1. 插件简介

Browser AI Assistant 是一个基于 Chrome Manifest V3 的浏览器侧边栏 AI 助手。它可以读取当前网页、多标签页、页面截图、图片附件和 DevTools Network 请求信息，并把这些上下文交给用户配置的大模型进行问答、总结、解释、改写、接口分析和故障排查。

插件的数据默认保存在本地浏览器 IndexedDB 中。模型渠道、API Key、聊天历史、Prompt 模板、页面提取规则和同步配置不会自动上传。只有用户显式开启同步后，数据才会备份到 Chrome Sync、WebDAV 或 S3 兼容存储。

## 2. 环境要求

使用插件前需要准备：

- Node.js 和 npm。
- Chrome、Edge 或其他支持 Manifest V3 与 Side Panel API 的 Chromium 浏览器。
- 至少一个可用的大模型服务端点。
- 对应模型服务的 API Key。

开发和验证常用命令：

```powershell
npm install
npm run build:extension
npm run typecheck
npm run test
npm run test:e2e
```

## 3. 本地安装插件

1. 在项目根目录构建扩展产物：

```powershell
npm run build:extension
```

2. 打开 Chrome 或 Edge 的扩展管理页面。
3. 开启“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择项目下的 `dist` 目录。

注意：不要选择项目根目录。插件清单文件会在构建后输出到 `dist/manifest.json`，浏览器加载目录必须是 `dist`。

构建后应至少能看到这些文件：

```text
dist/manifest.json
dist/index.html
dist/background/index.js
dist/content/index.js
```

## 4. 打开侧边栏

插件支持三种入口：

- 点击浏览器工具栏中的扩展图标。
- 在网页中打开右键菜单，点击“打开 AI 助手”。
- 使用快捷键 `Ctrl+Shift+Y`，macOS 使用 `Command+Shift+Y`。

打开侧边栏时，后台会记录当前业务标签页 ID。这个绑定会用于页面上下文提取和 DevTools Network 上下文匹配，避免误把插件页面或其他标签页作为分析对象。

## 5. 首次配置模型

首次使用前，需要先配置模型渠道和模型。

1. 打开侧边栏。
2. 进入“设置”。
3. 在渠道管理中新增渠道。
4. 选择端点类型。
5. 填写端点地址和 API Key。
6. 拉取远端模型列表，或手动新增模型。
7. 可选：测试模型连通性。
8. 可选：设置默认对话模型。
9. 可选：设置 AI 标题生成模型。
10. 如果模型支持图片理解，勾选视觉能力。

当前支持的端点类型：

- `openai_chat`：OpenAI Chat Completions 兼容协议。
- `anthropic_messages`：Anthropic Messages 协议。

模型级配置包括：

- 展示名称。
- 真实模型 ID。
- 温度参数。
- 最大 token 数。
- `topK`。
- 系统提示词。
- 是否支持视觉理解。
- 是否作为标题生成模型。

API Key 属于敏感信息，不应提交到 Git 仓库，也不应出现在截图、日志或教程示例中。

## 6. 发起普通网页对话

常规使用流程：

1. 打开需要分析的网页。
2. 打开插件侧边栏。
3. 确认当前模型已选择。
4. 根据需要打开或关闭“注入页面上下文”。
5. 输入问题。
6. 点击发送，或使用配置的发送快捷键。

默认情况下，新会话首问会把当前网页上下文注入系统提示词。继续追问、重新生成或编辑后重新生成时，不会重新注入当前页面状态，避免后续页面变化污染历史语义。

插件支持流式回复。流式模式下，AI 回复会逐段显示。如果流式连接在未收到任何内容前断开，插件会自动回退为非流式请求。

## 7. 页面上下文提取

插件可以提取当前页面内容作为对话上下文。上下文结构固定包含：

- 页面标题。
- 当前 URL。
- 页面内容。

提取模式有两种：

- 文本模式：提取页面可见文本。
- HTML 模式：提取完整 HTML 或命中节点的 HTML。

文本模式会跳过脚本、样式、模板、隐藏节点和 `aria-hidden` 内容。HTML 模式适合分析页面结构、表单、DOM 片段或需要查看源码的场景。

如果网页内容很长，插件会根据当前模型的 `maxTokens` 预算裁剪上下文。裁剪优先保证系统提示词、历史消息、思考过程和用户输入不会被页面内容挤掉。

## 8. 多标签页上下文

插件支持选择多个标签页作为上下文来源。

使用方式：

1. 打开上下文选择弹窗。
2. 加载当前窗口可选标签页。
3. 勾选需要参与分析的标签页。
4. 发送问题。

多标签页上下文会按用户选择顺序分别提取并合并。每个标签页仍保持标题、URL、内容的稳定结构。

多标签页提取允许部分失败。成功项会继续参与注入，失败项会在弹窗中显示中文错误；如果全部失败，则按空上下文处理，不阻断发送。

## 9. 页面提取规则

对于结构复杂的网站，可以配置页面提取规则，让插件优先抽取关键区域。

规则包含：

- 规则名称。
- URL 正则。
- 多行 CSS 选择器或 XPath。
- 排序顺序。

执行逻辑：

1. 按 `sortOrder` 匹配当前 URL。
2. 使用第一条命中的规则。
3. 按选择器顺序提取内容。
4. 如果选择器异常、提取为空或规则未命中，回退到全局页面提取。

保存规则前，插件会校验：

- URL 正则是否合法。
- CSS 选择器是否合法。
- XPath 是否合法。

插件也支持调用已配置模型，为当前 URL 生成候选 JavaScript RegExp。生成结果仍会经过合法性校验和数量限制。

## 10. 图片与视觉模型

当当前模型支持视觉理解时，插件支持：

- 上传图片。
- 粘贴图片。
- 截取当前标签页可见区域。
- 发送纯图片消息。
- 图片缩略图展示。
- 图片放大预览。
- 删除待发送图片。
- 在历史消息中重新加载图片附件。

图片限制：

- 类型仅支持 PNG、JPEG、WebP、GIF。
- 单张最大 5MB。
- 单条消息最多 5 张。

如果当前模型不支持视觉理解，插件会禁用上传、粘贴和截图入口，并在状态层拒绝带图请求。

协议差异：

- OpenAI 兼容协议会把图片转换为 `image_url` 内容块。
- Anthropic 协议会把图片转换为 base64 image 内容块。

## 11. 工具调用设置

工具调用用于让支持原生工具调用的模型按需请求插件提供的受控能力。当前版本提供 OpenAI Compatible 与 Anthropic 的工具调用基础、总开关、单工具启用列表和 `tavily_search` 网络搜索工具，不会自动控制浏览器。

使用方式：

1. 进入“设置”。
2. 打开“聊天偏好”。
3. 按需开启“启用工具调用”总开关。
4. 在工具列表中启用 `tavily_search`，模型会按需自行决定是否搜索。
5. 在聊天输入区 `.composer-actions` 的“工具调用”图标弹窗中，可以为当前聊天选择“启用”“启用全部”或“关闭”，也可以逐项切换单个工具。
6. 工具弹窗默认与输入区工具图标中心线对齐，靠近边缘时会自动收进视口内。
7. “当前聊天设置”抽屉不承载工具调用配置，避免和输入区工具弹窗形成重复入口。

Tavily 搜索说明：

- Tavily 搜索只通过 `tavily_search` 工具调用触发，不再提供独立网络搜索开关或“首次/每次对话”的搜索时机设置。
- Tavily API Key、综合答案、原始内容和最大结果数只在全局 Tavily 搜索工具配置中维护，当前聊天设置不再覆盖这些参数。
- 工具调用开始后，聊天消息列表中会先出现居中的灰色工具调用记录；工具返回后该记录可点击查看参数、耗时、结果摘要和关联附件。
- 工具搜索结果会作为 assistant 消息下方的网络搜索附件保存、展示、参与导出，并在后续对话中继续作为历史搜索上下文注入模型请求。
- 后续新增工具也应复用同一套调用记录和 `.message-xxxx-attachment` 附件展示方式。

安全边界：

- 工具调用默认关闭。
- 未注册或未启用的工具不会执行。
- 模型返回的工具参数会按外部输入处理，参数非法时会拒绝执行。
- 当前 DevTools Network 相关性筛选使用的是内部结构化输出能力，不受工具调用总开关影响。

## 12. DevTools Network 上下文

Network 上下文用于分析接口请求、响应、错误状态、请求体和响应体。该功能默认关闭，必须用户显式开启。

使用前需要：

1. 打开目标网页。
2. 打开该网页的 DevTools。
3. 切换到 Network 面板。
4. 刷新页面，让 DevTools 采集请求。
5. 打开插件侧边栏。
6. 开启 Network 上下文开关。
7. 输入分析需求并发送。

处理流程：

1. Side Panel 请求当前标签页的 Network 快照。
2. 根据用户需求和请求元数据，让模型筛选相关请求。
3. 根据筛选结果读取请求详情。
4. 对 URL、Header、Body 进行脱敏。
5. 把脱敏后的详情注入正式模型请求。
6. 将 Network 附件挂到本轮 assistant 消息下方。

筛选阶段只使用请求元数据，不写入聊天历史。正式分析阶段才注入完整详情。

Network 类型过滤支持：

- All
- Fetch/XHR
- Doc
- CSS
- JS
- Font
- Img
- Media
- Manifest
- WS
- Wasm
- Other

敏感字段会默认脱敏，例如：

- Authorization
- Cookie
- Token
- API Key
- Secret
- Password
- Session

如果 DevTools 未连接、页面正在刷新、没有请求、筛选为空或详情读取失败，插件会给出中文提示，并避免发送空详情分析请求。

## 13. Prompt 模板

Prompt 模板适合保存常用指令，例如“总结页面重点”“分析接口错误”“改写成更清晰的产品文案”等。

使用方式：

1. 在设置页维护 Prompt 模板。
2. 在聊天输入框中输入 `/`。
3. 从候选列表选择模板。
4. 补充用户问题并发送。

发送消息时，插件会保存：

- 用户可见输入。
- Prompt 调用快照。

请求模型时才会把 Prompt 快照展开到用户消息中。因此，后续修改模板不会改变历史消息的含义。

## 14. 会话历史与文件夹

插件使用 IndexedDB 保存聊天历史。

支持能力：

- 新建会话。
- 重命名会话。
- 删除会话。
- 归档会话。
- 创建文件夹。
- 重命名文件夹。
- 拖拽会话到文件夹。
- 保存每个会话的当前模型。
- 保存图片附件。
- 保存思考过程。
- 保存 Prompt 调用快照。
- 保存 Network 附件。

会话模型选择优先级：

1. 会话保存的模型。
2. 当前有效选择。
3. 默认对话模型。
4. 首个可用模型。

删除模型或渠道时，插件会清理无效默认模型和会话模型，避免刷新后残留不可用配置。

## 15. 隐私模式

隐私模式用于临时对话。

隐私模式规则：

- 消息只保存在内存态。
- 不自动写入 IndexedDB。
- 不进入同步快照。
- 可以手动保存为普通历史会话。
- 有消息时切换历史会话需要确认。

适合场景：

- 临时分析敏感页面。
- 不希望留下历史记录的排查。
- 测试模型或 Prompt。

## 16. AI 标题生成

如果设置了标题生成模型，插件会在首轮对话完成后尝试生成会话标题。

标题生成会参考：

- 用户输入。
- 页面上下文。
- AI 回复内容。

如果没有配置标题模型，或标题生成失败，插件会保留默认标题，不影响正常聊天。

## 17. 聊天记录导出

插件支持导出当前会话：

- Markdown
- Word
- PDF

导出内容包括：

- 会话标题。
- 导出时间。
- 会话时间。
- 消息数量。
- 消息角色。
- 正文。
- 思考过程。
- Prompt 快照。
- Network 附件详情。

导出安全机制：

- Markdown 会处理正文中的反引号围栏，避免提前截断。
- PDF 打印 HTML 会转义标题和正文，避免注入。
- 下载文件名会清理非法字符、控制字符、空标题和前导点。
- Blob URL 会在下载后释放。

## 18. 数据同步与备份

同步默认关闭。开启同步不等于开启自动同步，自动备份需要用户显式开启。

支持的备份目标：

- Chrome Sync
- WebDAV
- S3 兼容存储

备份能力：

- 手动备份。
- 手动恢复。
- 自动定时备份。
- 多份历史备份保留。
- 最大备份数量限制。
- 可选 AES-GCM 加密。

恢复规则：

- 手动恢复会覆盖本地业务数据。
- 恢复前必须二次确认。
- 恢复远端快照时保留本地密钥和远程凭据。
- 加密备份需要本地密钥正确才能恢复。

不会进入同步快照的敏感信息：

- 加密密钥。
- WebDAV 密码。
- S3 Secret Key。

备份前缀要求：

- 不能为空。
- 不能包含路径分隔符。
- 不能包含连续点号。

## 19. 常见问题

### 19.1 加载扩展时提示找不到清单文件

原因通常是加载了项目根目录，而不是构建后的 `dist` 目录。

解决方式：

1. 执行 `npm run build:extension`。
2. 在扩展管理页选择 `dist` 目录。

### 19.2 页面上下文提取失败

可能原因：

- 当前页面是浏览器受限页面。
- 页面尚未完成加载。
- 扩展刚重载，旧页面没有当前版本 Content Script。
- 目标标签页不是当前绑定业务标签页。

处理方式：

- 刷新页面。
- 重新打开侧边栏。
- 避免在 `chrome://` 等受限页面使用。

### 19.3 Network 上下文提示未连接 DevTools

处理方式：

1. 关闭当前页面 DevTools。
2. 重新打开当前页面 DevTools。
3. 切换到 Network 面板。
4. 刷新页面。
5. 等待插件显示已采集请求。

### 19.4 图片入口不可用

原因通常是当前模型没有标记为支持视觉理解。

处理方式：

1. 进入模型设置。
2. 找到当前模型。
3. 勾选支持视觉理解。
4. 回到聊天页重新选择该模型。

### 19.5 同步恢复失败

可能原因：

- 备份格式无效。
- provider 或 prefix 不匹配。
- 加密密钥错误。
- 远程凭据不可用。
- Chrome Sync 配额超限。

处理方式：

- 检查同步目标配置。
- 检查备份前缀。
- 确认加密密钥。
- 对 Chrome Sync 大数据备份，优先减少历史记录或改用 WebDAV/S3。

## 20. 安全注意事项

- 不要提交 API Key、访问令牌、密码、证书、私钥或连接串。
- 不要在截图中展示真实 API Key、S3 Secret Key、私有端点或个人账号。
- 默认不信任网页内容、模型返回、用户填写 URL、正则、CSS、XPath 和远程备份。
- Network 请求详情展示、导出和再次注入模型前都必须重新脱敏。
- 手动恢复远端备份会覆盖本地业务数据，必须确认备份来源可信。

## 21. 维护者检查清单

修改项目功能后，维护者应检查本教程是否需要同步更新。尤其是以下变更：

- 新增或删除入口、按钮、菜单、快捷键。
- 调整模型协议、模型参数或模型配置流程。
- 调整页面上下文、多标签页上下文或提取规则。
- 调整图片、截图或视觉模型能力。
- 调整 DevTools Network 上下文流程。
- 调整 Prompt 模板和斜杠调用。
- 调整会话历史、隐私模式、文件夹或标题生成。
- 调整导出格式、导出内容或文件名规则。
- 调整同步 provider、加密、备份、恢复或快照结构。
- 调整安装、构建、测试或排障流程。

如果代码变更影响用户可见能力、配置路径、操作步骤、安全边界或验证方式，应在代码提交后重新完善本文档。
